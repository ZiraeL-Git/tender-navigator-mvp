## Качество разбора
Общая оценка: `низкое качество на реальном кейсе`.

Что видно по факту:
- основной документ закупки пришел в формате `.doc`, а текущий MVP его не читает;
- PDF не дал текстового слоя текущему extractor'у;
- протоколы в `.zip` содержат JPG-сканы, а OCR для изображений и архивов пока не подключен;
- из всего пакета реально отработал только один `.docx`-протокол, и из него появился ложный `customer_name`.

## Практический вывод
Этот кейс хорошо подходит как контрольный benchmark для следующих улучшений:
- поддержка `.doc` или автоматическая конвертация в `.docx`;
- OCR fallback для PDF без текстового слоя;
- распаковка архивов и анализ вложенных файлов;
- OCR для JPG/PNG-сканов;
- ужесточение extractor'а `customer_name`, чтобы убрать ложные срабатывания.
